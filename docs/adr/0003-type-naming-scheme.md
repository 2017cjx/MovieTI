# 0003: タイプの見出し名はMBTI式のアーキタイプ名（監督マッチングは2026-07-14に撤回・スコープ外）

## Status
Accepted (2026-07-14)

## Context
16タイプの命名方法として、当初「あなたのタイプはスティーブン・スピルバーグです」のように
監督名をそのまま見出しにする案を検討した。TMDbの監督フィルモグラフィーを実データで
スコアリングし、各タイプに一番フィットする監督を機械的にマッチングする仕組み自体は
データに裏付けられた面白い設計だが、監督名を見出しに使うと以下の懸念がある。

- 大衆的なタイプは有名監督にマッチしやすいが、ニッチなタイプほど一般には無名の監督に
  マッチする。当初の要件「とっつきやすさ」と相性が悪い場合がある。
- 「あなたは〇〇監督です」という見出しは、監督を知らないと意味が伝わらない
  （MBTIの「提唱者」のような、説明不要の言葉と比べて理解のハードルが上がる）。

## Decision
役割を分離する。

- **タイプの見出し名**: MBTIに倣い、説明不要のアーキタイプ名にする
  （具体的な16個の名称はコンテンツ設計フェーズで別途詰める）。
  2026-07-14追記: アプリの言語を英語に確定したため（CONTEXT.md「アプリの言語」参照）、
  タイプ名は英語（例: MBTIの"Advocate"のような形式）にする。
- **監督マッチング**: 当初、見出しとは独立した結果本文中の裏付け・パーソナライズ要素として
  残す方針にしていた（50人程度の監督フィルモグラフィーを同じ4軸でスコアリングし、
  ユーザーの軸スコアに一番近い監督を都度マッチングする案）。

  **2026-07-14追記: この監督マッチング機能は撤回し、スコープ外とする。** 理由:
  候補監督50人のリストアップ、TMDb `person`フィルモグラフィー取得、軸→監督指標の変換
  ロジックの実装・検証という、残り日数に対してスコープが大きすぎる作業が必要になることが
  判明したため。タイプの見出し名（英語のアーキタイプ名）と説明文だけで結果画面の
  パーソナライズ要素は成立させる（[[パーソナライズ層]]はシグネチャー映画・実数の統計・
  ライブLLM演出に限定）。

## Consequences
- 16タイプ名自体の作詞・コピーライティングは別タスクとして残る（未着手）。
- 監督データ（`directors.json`）関連の設計時作業・API設計は不要になった。SPEC.mdから
  該当箇所を削除済み。
