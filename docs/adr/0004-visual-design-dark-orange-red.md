# 0004: ビジュアルデザインはダークモード基調・オレンジ〜レッドのグラデーション（紫は使わない）

## Status
Accepted (2026-07-14)

## Context
`src/index.css`の初期スキャフォールド（Viteテンプレート由来）はアクセントカラーに紫
（ライトモード`#aa3bff`、ダークモード`#c084fc`）を使っていた。紫のグラデーションは
生成AIツール製のUIで頻出する配色で、「いかにもAIが作った感」が出やすく、[[スコープ境界]]で
確認した「一般論にせず具体的で刺さる内容にする」という診断結果の品質基準と同じ理由で、
見た目についても避けたいというフィードバックを受けた。

## Decision
- 配色はダークモードを既定（かつ唯一）のテーマとする。OS設定に応じたライト/ダーク切り替え
  （`prefers-color-scheme`）は行わない — これはAdaptiveなユーティリティ配色ではなく、
  MovieTIというアプリ自体の見た目のアイデンティティとして固定する。
- アクセントカラーは紫を一切使わず、オレンジ（`#f97316`）→レッド（`#ef4444`）の
  グラデーションを基調とする（`--accent-gradient`としてCSS変数化）。背景は暖色寄りの
  ほぼ黒（`#130e0b`）とし、青みがかったダーク（Viteデフォルトの`#16171d`のような）
  ではなく、アクセントのオレンジ〜レッドと馴染む暖色系のダークにする。
- 見出し（`h1`）にはグラデーションをテキストに直接適用する（`background-clip: text`）。
  ボタンやハイライトなど今後追加するUIコンポーネントも、紫を使わずこの
  `--accent` / `--accent-2` / `--accent-gradient`を再利用する。

## Consequences
- 今後追加するすべてのUIコンポーネント（星評価、進捗バー、レーダーチャート等）は
  この配色トークンに従う。個別に色を決め打ちしない。
- レーダーチャート（recharts等、5節）の配色もこのオレンジ〜レッド系に揃える必要がある
  （未着手、実装時に対応）。
- 「AIジェネレーテッドっぽさを避ける」という要望は配色だけでなく今後のUI全体（レイアウト、
  タイポグラフィ、コピー）にも適用されるべき指針だが、具体的なUIコンポーネントの
  デザインはまだ未着手（SPEC.md 9節）。
